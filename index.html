Lecture 1

Methods (Functions)

What is a Method?
🌍 Real World

Imagine you're a restaurant manager. Instead of cooking every dish yourself, you hire specialist chefs: a pizza chef, a pasta chef, a dessert chef. You call on them when needed. Methods are exactly those specialist chefs — each does one specific job, and you call them when you need that job done.

☕ Java

A method is a named block of code that performs a specific task. You write it once and can call it many times. This keeps your code organized and avoids repetition.

Method Syntax:

returnType methodName(parameterList) { ... return value; }

• returnType: what type of value the method gives back. Use void if it gives nothing back.

• methodName: what you call the method.

• parameterList: inputs the method needs (can be empty).

• return: sends a value back to whoever called the method.

Example 1 void method — no return value, no parameters
DrawLine.java
public class Main {
    public static void main(String[] args) {
        drawLine(); // call the method
        System.out.println("Java Lectures");
        drawLine(); // call again — reuse!
    }

    public static void drawLine() {
        for (int i = 1; i <= 10; i++) {
            System.out.print("*");
        }
        System.out.println();
    }
}
Output
**********
Java Lectures
**********
Example 2 Method with parameters
Params.java
public static void drawLine(char symbol, int length) {
    for (int i = 1; i <= length; i++) {

public static int sum(int i, int j) {

Variable Scope — Local vs Global
🌍 Real World

Think of your house (the class) and your bedroom (a method). A key inside your bedroom only opens your bedroom door — that's a local variable. A master key that works anywhere in the house — that's a global variable (called a field in Java).

Local Variable: Declared inside a method. Only visible inside that method. Dies when the method finishes.

Global (static) Variable: Declared outside all methods (but inside the class) with static. Accessible by all methods in the class.

Scope.java
public class Main {
    static int x; // global — accessible everywhere

    public static void main(String[] args) {
        x = 100;
        System.out.println("x = " + x); // 100
        display();
    }

    public static void display() {
        System.out.println("x = " + x); // also 100
    }
}
Method Overloading
🌍 Real World

Think of the word "play". You can play football, play piano, play a video game. Same word — but different actions depending on context (what you're playing). Method overloading is the same: same method name, but different parameters.

Java allows multiple methods with the same name as long as their parameter lists are different (different types or different count). The return type alone is NOT enough to distinguish them.

A method signature = method name + parameter list (NOT return type).

Overloading.java
public static int    getlarger(int a, int b) { ... }
public static int    getlarger(int a, int b, int c) { ... }
public static double getlarger(double x, double y) { ... }
// Java picks the right one based on what arguments you pass
Calling getlarger(20, 30)
→ uses first version → returns 30
Calling getlarger(20, 30, 70)
→ uses second version → returns 70
Call by Value vs Call by Reference
🌍 Real World

Call by Value: You photocopy a document and give it to someone. They can write on the copy all they want — your original is unchanged.
Call by Reference: You give someone your actual document. Changes they make affect the original.

• Primitive types (int, double, char, boolean…): passed by value. The method gets a copy. Changes inside don't affect the original variable.

• Arrays and Objects: passed by reference. The method gets the memory address. Changes inside DO affect the original.

Example Array passed by reference
ByRef.java
public static void increase(int[] a) {
    for (int i = 0; i < a.length; i++) {
        a[i] = a[i] * 2; // modifies original!
    }
}
// In main:
int[] iq = {126, 167, 95};
increase(iq);
System.out.println(Arrays.toString(iq));
Output
[252, 334, 190]

Lecture 2

Object-Oriented Programming

What is OOP?
🌍 Real World

Before OOP: imagine building a city where every single building is custom-built from scratch with no plans. OOP is like having blueprints. One blueprint (class) lets you build many identical houses (objects), each with their own address and furniture (data), but all shaped the same way.

☕ Java

OOP is a style of programming where you model your program as interacting objects. Each object has data (fields/attributes) and behaviors (methods/operations).

Class = the blueprint. Defines what data and methods objects will have.

Object = an actual instance created from the blueprint. Lives in memory.

Field = data stored inside an object (e.g., name, age, salary).

Method = action the object can perform (e.g., getSalary(), setName()).

Writing a Class
🌍 Real World

Let's design a Car blueprint. Every car has: color, speed, fuel level (data). Every car can: accelerate, brake, refuel (methods). The blueprint doesn't drive anywhere — but when you actually build a car from it, THAT car can drive.

Example 1 Creating a Car class and objects
Car.java
public class Car {
    private String maker;   // field (data)
    private int model;       // field (data)

    // SETTER — sets data
    public void setMaker(String m) { maker = m; }
    public void setModel(int year)  { model = year; }

    // GETTER — reads data
    public String getMaker() { return maker; }
    public int    getModel() { return model; }
}
Main.java
Car c1 = new Car();   // create object
c1.setMaker("Honda");
c1.setModel(2016);
System.out.println(c1.getMaker());  // Honda

Car c2 = new Car();   // separate object
c2.setMaker("Toyota");
c2.setModel(2019);
Output
Honda
Access Modifiers — public vs private
🌍 Real World

A bank vault has two areas: the public lobby where anyone can go, and the private vault where only authorized staff access the money directly. The teller (method) is public — you interact with them. The vault contents (data) are private — the teller accesses them on your behalf.

private: Only accessible within the same class. Fields are almost always private to protect data.

public: Accessible from anywhere — from Main, from other classes, etc.

⚠️

If a field is private and you try to access it directly from another class (e.g., r1.length = 60), Java gives a compile error: "length has private access in Rectangle". You MUST use a public setter method.

Encapsulation (Data Hiding)
🌍 Real World

A TV remote. You press buttons (public methods) to change channel or volume. You never touch the circuit board inside (private fields). The TV hides its internal wiring — that's encapsulation. You only interact through the provided interface (the remote).

Encapsulation = keeping fields private + providing public getters and setters to access/modify them. This lets the class validate data before accepting it.

Example 2 Data validation in a setter
Validation.java
public void setMaker(String m) {
    if (m.equals("Toyota") || m.equals("Honda") || m.equals("Mercedes")) {
        maker = m;
    } else {
        System.out.println("invalid maker");
    }
}

public void setModel(int year) {
    if (year > 2012) {
        model = year;
    } else {
        System.out.println("invalid model");
    }
}
Example 3 Full Rectangle class with encapsulation
Rectangle.java
public class Rectangle {
    private double length;
    private double width;

    public void setLength(double l) { length = l; }
    public void setWidth(double w)  { width = w;  }
    public double getLength() { return length; }
    public double getWidth()  { return width;  }
    public double getArea()   { return length * width; }
}

// Main:
Rectangle r1 = new Rectangle();
r1.setLength(10);
r1.setWidth(12.5);
System.out.println(r1.getArea()); // 125.0
Output

The this Keyword
🌍 Real World

When someone asks "whose car is this?" — you say "this is MY car." In Java, this refers to the current object — it's the object saying "I own this field."

Java automatically adds this. before instance variable references inside instance methods. It's needed explicitly when a local parameter has the same name as an instance field (variable hiding).

ThisKeyword.java
class Circle {
    public int radius;

    void setRadius(int radius) {
        // 'radius' alone refers to the parameter
        // 'this.radius' refers to the field
        this.radius = radius;
    }
}

Lecture — Constructors

Constructors

What is a Constructor?
🌍 Real World

When a new employee joins a company, there's an onboarding process: they get a badge, assigned a desk, a computer set up. This happens automatically the moment they join. A constructor is exactly that — it runs automatically when a new object is created, setting everything up.

Key rules for constructors:

• Same name as the class

• No return type (not even void)

• Called automatically with new

• Can be overloaded (multiple constructors with different parameters)

Example 1 Basic constructor
Constructor.java
public class Rectangle {
    private double length;
    private double width;

    public Rectangle() {          // constructor
        length = 10;               // auto-runs on new Rectangle()
        width  = 15;
    }
    public double getLength() { return length; }
    public double getWidth()  { return width; }
}

// Main:
Rectangle r1 = new Rectangle(); // constructor runs here
System.out.println(r1.getLength()); // 10.0
System.out.println(r1.getWidth());  // 15.0
Output
10.0
15.0
Example 2 Constructor overloading
ConstructorOverload.java
public class Rectangle {
    private double length, width;

    public Rectangle() {                     // no-arg constructor
        length = 10; width = 15;
    }

    public Rectangle(double l, double w) { // parameterized constructor
        length = l; width = w;
    }
    public double getLength() { return length; }
    public double getWidth()  { return width; }
}

// Main:
Rectangle r1 = new Rectangle();        // 10, 15
Rectangle r2 = new Rectangle(40, 60); // 40, 60
System.out.println(r1.getLength());   // 10.0
System.out.println(r2.getLength());   // 40.0
Output
10.0
40.0
The Default Constructor

If you write no constructor at all, Java provides a default constructor automatically. It initializes:

• Numeric fields → 0 (int, double, etc.)

• boolean fields → false

• String and object fields → null

⚠️ If you write any constructor, Java no longer provides the default. You must write it yourself if needed.

Lecture — Static

Static Class Members

Static Fields and Static Methods
🌍 Real World

A school has students (objects). Each student has their own name and grade (instance fields). But there's ONE school name, ONE principal — shared by all students. That shared thing is a static field. You don't ask "what's Student Ahmed's school name?" — you say "what's the school's name?"

Static fields: Belong to the CLASS itself, not to any object. All objects share the same copy.

Static methods: Called using the class name, not an object. Example: Math.sqrt(25) — you don't create a Math object first.

Static methods cannot directly access instance (non-static) fields. They can only use static fields.

Example 1 Counting objects with a static field
StaticField.java
public class Car {
    private String maker;
    private int model;
    private static int no_of_objects; // shared across ALL cars

    public Car() {
        maker = "Honda";
        model = 2017;
        no_of_objects++;  // increments every time new Car is made
    }

    public static int get_no_of_objects() {
        return no_of_objects;
    }
}

// Main:
Car c1 = new Car();
Car c2 = new Car();
Car c3 = new Car();
Car c4 = new Car();

// Both work — static method can be called on class OR object:
System.out.println(Car.get_no_of_objects()); // 4
System.out.println(c2.get_no_of_objects());  // also 4
Output
4
4
Example 2 Utility class with all static methods
Calc.java + Main.java
public class Calc {
    public static int    add(int a, int b) { return a + b; }
    public static int    subtract(int a, int b) { return a - b; }
    public static int    multiply(int a, int b) { return a * b; }
}

// Main — no object needed, call with class name:
System.out.println(Calc.add(30, 20));      // 50
System.out.println(Calc.subtract(30, 20)); // 10
System.out.println(Calc.multiply(30, 20)); // 600
Output
50
10
600
